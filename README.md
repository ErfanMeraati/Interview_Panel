# پرتال استخدام آنلاین (Airex HR Portal)

پرتال درخواست استخدام آنلاین به زبان فارسی (راست‌چین) با تقویم شمسی — شامل فرم عمومی برای متقاضیان شغلی و پنل مدیریتی برای بررسی، مصاحبه‌گذاری و ارسال پیامک به متقاضیان.

## امکانات

**صفحه‌ی متقاضی (عمومی)**
- فرم درخواست استخدام کاملاً فارسی و راست‌چین با تاریخ تولد شمسی
- آپلود تصویر پروفایل و فایل رزومه (PDF / Word)
- اعتبارسنجی سمت کاربر برای فیلدهای اجباری، فرمت ایمیل و شماره تماس
- طراحی اختصاصی با پالت رنگی برند (`#c40442` / `#253289` / سفید)

**پنل مدیریت (ادمین)**
- ورود با رمز عبور
- داشبورد با آمار کلی (کل درخواست‌ها، جدید، در حال بررسی، پذیرفته‌شده)
- فیلتر بر اساس وضعیت + جستجوی نام/موقعیت شغلی
- باکس «مصاحبه‌های امروز» با لینک مستقیم به پروفایل هر متقاضی
- صفحه‌ی پروفایل هر متقاضی: مشاهده‌ی کامل اطلاعات، دانلود رزومه، تغییر وضعیت بررسی، یادداشت داخلی
- **دعوت به مصاحبه**: انتخاب تاریخ (شمسی) و ساعت + ارسال خودکار پیامک به متقاضی از طریق کاوه‌نگار
- **حذف پرونده**: حذف کامل رکورد از دیتابیس + حذف فایل عکس/رزومه از سرور، با تاییدیه‌ی اجباری

## تکنولوژی‌ها

- PHP (بدون فریم‌ورک خاص) + PDO
- MySQL / MariaDB
- HTML / CSS / JavaScript خالص (بدون نیاز به build tool)
- فونت [Vazirmatn](https://github.com/rastikerdar/vazirmatn) برای متن فارسی
- [کاوه‌نگار](https://kavenegar.com) برای ارسال پیامک

## ساختار پروژه

```
/
├── index.php                  # فرم عمومی درخواست استخدام
├── functions.php               # توابع عمومی (e(), require_admin() و ...)
├── db.php                       # اتصال PDO به دیتابیس
├── jalali.php                   # توابع کمکی تاریخ شمسی برای نمایش عمومی
├── assets/
│   └── main.css                 # استایل مشترک صفحات
├── uploads/
│   ├── photos/                  # تصاویر پروفایل آپلودشده
│   └── resumes/                 # فایل‌های رزومه آپلودشده
└── panel/                        # پنل مدیریت (ادمین)
    ├── dashboard.php             # لیست و آمار درخواست‌ها + مصاحبه‌های امروز
    ├── profile.php                # پروفایل کامل هر متقاضی
    ├── delete.php                 # حذف پرونده (DB + فایل‌ها)
    ├── interview.php              # ثبت زمان مصاحبه + ارسال پیامک
    ├── interview_helpers.php      # تبدیل دوطرفه تاریخ شمسی/میلادی
    ├── sms_config.php             # تنظیمات کاوه‌نگار (API Key، نام الگو) — حاوی اطلاعات محرمانه
    ├── sms_helper.php              # توابع ارسال پیامک (Lookup API کاوه‌نگار)
    └── logout.php                  # خروج از پنل ادمین
```

> ساختار دقیق فایل‌های `functions.php`, `db.php`, `jalali.php`, `index.php` و `assets/main.css` ممکن است بسته به نسخه‌ی نهایی شما کمی متفاوت باشد؛ لیست بالا را با فایل‌های واقعی پروژه‌تان تطبیق دهید.

## پیش‌نیازها

- هاست/سرور با پشتیبانی از PHP 8+ و افزونه‌ی `curl`
- دیتابیس MySQL 5.7+ یا MariaDB
- دسترسی phpMyAdmin یا هر ابزار مدیریت دیتابیس دیگر
- یک حساب فعال در [کاوه‌نگار](https://kavenegar.com) برای ارسال پیامک

## نصب و راه‌اندازی

### ۱. آپلود فایل‌ها

تمام فایل‌های پروژه را در روت (یا ساب‌دامین/پوشه‌ی موردنظر) هاست خود آپلود کنید.

### ۲. ساخت دیتابیس

یک دیتابیس جدید بسازید و جدول زیر را در آن ایجاد کنید:

```sql
CREATE TABLE `applications` (
  `id` int NOT NULL AUTO_INCREMENT,
  `full_name` varchar(190) COLLATE utf8mb4_persian_ci NOT NULL,
  `phone` varchar(30) COLLATE utf8mb4_persian_ci NOT NULL,
  `email` varchar(190) COLLATE utf8mb4_persian_ci NOT NULL,
  `national_id` varchar(20) COLLATE utf8mb4_persian_ci DEFAULT NULL,
  `city` varchar(100) COLLATE utf8mb4_persian_ci DEFAULT NULL,
  `birth_jalali` varchar(30) COLLATE utf8mb4_persian_ci DEFAULT NULL,
  `position` varchar(190) COLLATE utf8mb4_persian_ci NOT NULL,
  `experience` varchar(60) COLLATE utf8mb4_persian_ci DEFAULT NULL,
  `message` text COLLATE utf8mb4_persian_ci,
  `photo_path` varchar(255) COLLATE utf8mb4_persian_ci DEFAULT NULL,
  `resume_path` varchar(255) COLLATE utf8mb4_persian_ci DEFAULT NULL,
  `resume_original_name` varchar(255) COLLATE utf8mb4_persian_ci DEFAULT NULL,
  `status` enum('جدید','در حال بررسی','پذیرفته شد','رد شد') COLLATE utf8mb4_persian_ci NOT NULL DEFAULT 'جدید',
  `admin_note` text COLLATE utf8mb4_persian_ci,
  `submitted_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `submitted_at_jalali` varchar(30) COLLATE utf8mb4_persian_ci DEFAULT NULL,
  `interview_date` date DEFAULT NULL,
  `interview_time` time DEFAULT NULL,
  `interview_status` enum('none','scheduled','done','cancelled') COLLATE utf8mb4_persian_ci DEFAULT 'none',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_persian_ci;
```

### ۳. تنظیم اتصال دیتابیس

در `db.php`، مشخصات اتصال (نام دیتابیس، یوزرنیم، پسورد) را با اطلاعات هاست خودتان جایگزین کنید.

### ۴. تنظیم رمز ورود پنل ادمین

در فایل مربوط به احراز هویت پنل (`functions.php` یا فایلی که `require_admin()` در آن تعریف شده)، رمز عبور پیش‌فرض را حتماً به یک رمز قوی و اختصاصی تغییر دهید.

### ۵. تنظیم پیامک (کاوه‌نگار)

فایل `panel/sms_config.php` را باز کنید و:

1. `KAVENEGAR_API_KEY` را با کلید واقعی حساب خودتان جایگزین کنید.
2. یک الگوی پیامکی (Pattern) در پنل کاوه‌نگار بسازید و برای تایید ارسال کنید. **نکته مهم:** الگوهای سرویس Verify/Lookup باید بیانگر «ثبت درخواست کاربر» باشند، نه یک اطلاع‌رسانی یک‌طرفه؛ نمونه‌ی تاییدشده:

   ```
   سلام %token10
   درخواست شما برای مصاحبه استخدامی ثبت شد
   تاریخ: %token20
   ساعت: %token3
   آدرس: [آدرس شرکت]
   تلفن: [تلفن شرکت]
   با احترام
   [نام شرکت]
   ```

3. محدودیت فاصله (Space) در توکن‌ها را رعایت کنید:

   | توکن | فاصله مجاز | کاربرد در این پروژه |
   |---|---|---|
   | `token`, `token2`, `token3` | خیر | مقادیر بدون فاصله (مثلاً ساعت) |
   | `token10` | تا ۵ فاصله | نام کامل متقاضی |
   | `token20` | تا ۸ فاصله | تاریخ شمسی (روز + نام ماه + سال) |

4. اسم الگو را در `KAVENEGAR_INTERVIEW_TEMPLATE` وارد کنید.

### ۶. پوشه‌های آپلود

مطمئن شوید پوشه‌های `uploads/photos` و `uploads/resumes` وجود دارند و PHP اجازه‌ی نوشتن (write) در آن‌ها را دارد (معمولاً پرمیشن `755` یا `775` کافی است).

## استفاده

- **لینک متقاضی**: آدرس اصلی سایت (`index.php`) را برای متقاضیان شغلی ارسال کنید.
- **لینک پنل ادمین**: از مسیر `panel/dashboard.php` وارد شوید (رمز عبور تنظیم‌شده در مرحله ۴ را وارد کنید).

## نکات امنیتی مهم

- **هرگز `panel/sms_config.php` را با کلید واقعی API در یک ریپازیتوری عمومی (Public) گیت‌هاب قرار ندهید.** پیشنهاد می‌شود:
  - این فایل را به `.gitignore` اضافه کنید.
  - به‌جای آن، یک فایل نمونه مثل `sms_config.example.php` (بدون مقدار واقعی) در ریپازیتوری قرار دهید.
- رمز عبور پنل ادمین را دوره‌ای تغییر دهید و از رمزهای پیش‌فرض/ساده استفاده نکنید.
- در صورتی که کلید API کاوه‌نگار تا به حال در جایی عمومی (چت، فایل به اشتراک‌گذاشته‌شده و ...) دیده شده، آن را از پنل کاوه‌نگار **غیرفعال و بازتولید (Regenerate)** کنید.
- فایل‌های آپلودشده (عکس/رزومه) در دسترس عمومی هستند اگر مسیرشان لو برود؛ در صورت نیاز به محرمانگی بیشتر، دسترسی مستقیم به پوشه‌ی `uploads` را از طریق `.htaccess` محدود کرده و فایل‌ها را فقط از طریق یک اسکریپت PHP احرازهویت‌شده تحویل دهید.
- ورودی‌های کاربر (خصوصاً در جستجو و فیلترها) با Prepared Statements پردازش می‌شوند؛ در صورت افزودن فیلد جدید، همین الگو را حفظ کنید.

## مشکلات رایج

| مشکل | راه‌حل احتمالی |
|---|---|
| ارسال درخواست از فرم متقاضی کار نمی‌کند | مطمئن شوید پروژه روی هاست واقعی (نه Artifact) اجرا می‌شود و اتصال دیتابیس در `db.php` درست است |
| پیامک ارسال نمی‌شود، خطای «ساختار کد صحیح نمی‌باشد» | بررسی کنید توکن‌های بدون‌فاصله (`token`, `token2`, `token3`) مقدار دارای فاصله نگرفته باشند؛ نام و تاریخ باید در `token10`/`token20` قرار بگیرند |
| الگوی پیامکی توسط کاوه‌نگار رد می‌شود | متن الگو باید بیانگر «ثبت درخواست کاربر» باشد، نه یک پیام تبلیغاتی/اطلاع‌رسانی صرف |
| آپلود فایل با خطا مواجه می‌شود | پرمیشن پوشه‌های `uploads/photos` و `uploads/resumes` و مقدار `upload_max_filesize` در PHP را بررسی کنید |

## توسعه‌های پیشنهادی آینده

- ارسال ایمیل خودکار به ادمین هنگام ثبت درخواست جدید
- خروجی گرفتن از لیست متقاضیان به‌صورت Excel/CSV
- تعریف چند سطح دسترسی برای ادمین‌ها (مدیر منابع انسانی، مصاحبه‌کننده و ...)
- ثبت تاریخچه‌ی تغییرات وضعیت هر پرونده (Audit Log)
